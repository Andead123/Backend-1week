1주차에 학습한 내용

웹이 무엇인지에 대해 배웠다. 인터넷은 전 세계 컴퓨터와 기기를 연결하는 거대한 글로벌
네트워크이고 웹이란 인터넷 위에서 동작하는 서비스 중 하나로 인터넷에 연결된 전 세계 사용자들이 서로의 정보를
공유할 수 있는 장소이다. 웹에서 client와 server가 상호작용할때 client가 정보를 요청하여 이에 답하여 보내는 역할이 백엔드,
화면 렌더링을 해 주는 것이 프론트엔드의 역할로 볼 수 있다. 
Http란 웹에서 데이터를 주고받는 서버-클라이언트 모델의 프로토콜로, 클라이언트의 요청(request)과 서버의 응답(response)을 통해 작동한다.
Http의 주요 상대로는  200 OK 201 Created 400 Bad Request 404 Not Found  500 Internal Server Error 가 있다.
API란  한 프로그램이 다른 프로그램의 기능이나 데이터를 사용할 수 있도록 미리 정해놓은 약속(규칙)이자 소통 창구이다.

3.온라인 쇼핑몰 프로젝트 API 명세서


회원 API


1.회원 등록 (가입)
기능 설명: 새로운 사용자의 정보를 받아 회원으로 등록합니다.
요청 방식: POST /api/members/join

요청 데이터:
{
  "username": "new_user",
  "password": "password123!",
  "email": "user@example.com",
  "address": "서울시 강남구"
}

응답 데이터:

{
  "memberId": 1,
  "username": "new_user",
  "message": "회원가입이 완료되었습니다."
}

2. 회원 목록 조회
기능 설명: 시스템에 등록된 모든 회원의 목록을 조회합니다.
요청 방식: GET /api/members

응답 데이터:


[
  {
    "memberId": 1,
    "username": "new_user",
    "email": "user@example.com"
  },
  {
    "memberId": 2,
    "username": "another_user",
    "email": "another@example.com"
  }
]

상품 API
1. 상품 정보 등록
기능 설명: 판매할 새로운 상품의 정보를 시스템에 등록합니다.

요청 방식: POST /api/products

요청 데이터:

JSON

{
  "productName": "반팔 티셔츠",
  "price": 18000,
  "stock": 100
}

응답 데이터:

{
  "productId": 101,
  "productName": "반팔 티셔츠",
  "message": "상품이 성공적으로 등록되었습니다."
}
2. 개별 상품 정보 상세 조회
기능 설명: 특정 상품 ID를 이용해 해당 상품의 상세 정보를 조회합니다.

요청 방식: GET /api/products/101

응답 데이터:

{
  "productId": 101,
  "productName": "반팔 티셔츠",
  "price": 18000,
  "stock": 100,
  "description": "부드러운 면 100% 소재로 제작된 티셔츠입니다."
}

주문 API
1. 주문 정보 생성 (상품 주문)
기능 설명: 특정 회원이 특정 상품을 주문합니다.

요청 방식: POST /api/orders (?)
요청 데이터:

{
  "memberId": 1,
  "productId": 101,
  "quantity": 2
}

응답 데이터:

{
  "orderId": 5001,
  "message": "주문이 성공적으로 처리되었습니다."
}
